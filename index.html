正则表达式

作用：对用户输入的内容进行校验

它的校验是专注于合法性校验

在正则眼中，所以内容都是字符串

学习正则主要就是学习它的元字符

元字符：就是一些特殊字符，在其它场景下它就是一个普通字符但是在正则中是有特殊含义的字符

正则的常用方法

  • 类型： RegExp,在js可以使用 //简写创建正则表达式对象
  • test()：正则表达式.test(你需要验证的字符串)，它返回一个bool，如果校验通过则返回true,否则返回false

正则的写法

  • 先占位后修饰
  • 从左到右写

元字符-占位符

占据一个位置，说明这个位置你需要添加相应的字符

  • . : 点，代表任意字符
  • \w:代表合法字符： A-Za-z0-9_ 掌握
  • \W:代表非法字符，除了合法的就是非法的 掌握
  • \d:代表数字 0-9 掌握
  • \D:代表非数字，除了0-9之外的都是非数字
  • \s:代表空字符：空格，制表位，换行…
  • \S:代表非空字符
  • []代表一个具体的范围，用于匹配这个范围中的某个字符 掌握

元字符-修饰符

它不能单独的存在

它的作用是用来修饰它前面的字符所出现的次数

  • * :代表前面的字符出现任意次，前面的字符默认只是前面的一个字符，如果需要修饰多则使用()包含
  • ？：修饰前面的字符出现0次或1次
  • +：修饰前面的字符出现1次或多次
  • {n}:修饰前面的字符出现n次
  • {n,m}:修饰前面的字符最少出现n次，最多出现m次
  • {n,}:修饰前面的字符最少出现n次，最多没有限制

元字符-其它字符

  • ^:它**开头
  • $:以**结尾，它们的组合一般用于需要严格限制位数的场合
  • -在[]中代表连贯的范围
  • ():可以改变优先级
  • \:转义符，将字符当成普通字符来处理
  • |：代表或者

项目优化

1, 项目从git拉下来之后安装包尽量使用yarn install 来下载所有的包,使用npm 或者cnpm 会出现包下载不全面或者下载包报错的情况,导致项目运行不起来

2, 项目打包体积过大可以使用 “compression-webpack-plugin”: “6.1.1” 第三方包来压缩包的体积 注意:这个压缩只有在打包上线的时候才会压缩,开发环境下运行项目不会压缩 在vue.config.js文件里面配置
